# VisualRedNoteGen: Multimodal Social Media Caption Generation

## 1. Project Overview

VisualRedNoteGen is a multimodal text generation project that generates social-media-style captions from images. The project uses a pretrained CLIP image encoder to extract visual features from an input image, and trains a GRU-based caption decoder to generate captions token by token.

The goal of this project is to explore whether visual information can improve caption generation compared with a text-only caption generator. This project satisfies the **Multimodal Modeling** extra criterion because the generated text is conditioned on image input.

## 2. Dataset

This project uses the **Instagram Images with Captions** dataset from Kaggle.

The raw dataset should have the following structure after downloading and unzipping:

```text
data/raw/
  instagram_data/
    captions_csv.csv
    img/
      insta1.jpg
      insta2.jpg
      ...
````

The CSV file contains image-caption pairs. In this dataset, the important columns are:

```text
Image File
Caption
```

The raw image files are not included in this GitHub repository because they are large. Please download the dataset separately from Kaggle.

## 3. Model Architecture

The main model is a CLIP-conditioned caption generator.

The architecture is:

```text
Input image
   ↓
Pretrained CLIP image encoder
   ↓
Image embedding
   ↓
Trainable projection layer
   ↓
Initial hidden state of GRU decoder
   ↓
Generated caption tokens
```

The CLIP image encoder is frozen during training. The trainable parts are:

1. Image-to-hidden projection layer
2. Token embedding layer
3. GRU caption decoder
4. Output vocabulary projection layer

The decoder is trained using next-token prediction with cross-entropy loss.

## 4. Extra Criterion

This project pursues the **Multimodal Modeling** extra criterion.

The model uses both image and text modalities:

* Image input: processed by CLIP image encoder
* Text output: generated by GRU decoder

The image representation directly conditions the caption decoder, so the model is not a text-only language model.

## 6. Reproducing the Project

### Step 1: Download the Dataset

Download the Instagram Images with Captions dataset from Kaggle and place it under:

```text
data/raw/instagram_data/
```

Expected files:

```text
data/raw/instagram_data/captions_csv.csv
data/raw/instagram_data/img/
```

### Step 2: Preprocess the Dataset

Run the preprocessing script:

```bash
python src/prepare_data.py \
  --csv data/raw/instagram_data/captions_csv.csv \
  --image-root data/raw/instagram_data \
  --image-col "Image File" \
  --caption-col "Caption"
```

This will create:

```text
data/processed/train.csv
data/processed/val.csv
data/processed/test.csv
data/processed/vocab.json
```

### Step 3: Train the CLIP-Conditioned Caption Model

For a quick local test:

```bash
python src/train_caption.py \
  --data-dir data/processed \
  --epochs 1 \
  --batch-size 4 \
  --lr 1e-3 \
  --clip-model-name openai/clip-vit-base-patch32
```

For a more complete training run:

```bash
python src/train_caption.py \
  --data-dir data/processed \
  --epochs 5 \
  --batch-size 16 \
  --lr 1e-3 \
  --clip-model-name openai/clip-vit-base-patch32
```

The trained checkpoints will be saved to:

```text
outputs/checkpoints/clip_caption_best.pt
outputs/checkpoints/clip_caption_last.pt
```

### Step 4: Evaluate the Model

```bash
python src/evaluate_caption.py \
  --data-dir data/processed \
  --split test \
  --checkpoint outputs/checkpoints/clip_caption_best.pt
```

The evaluation reports test loss and perplexity.

### Step 5: Generate a Caption for One Image

Example:

```bash
python src/generate_caption.py \
  --image data/raw/instagram_data/img/insta11974.jpg \
  --checkpoint outputs/checkpoints/clip_caption_best.pt \
  --min-len 5 \
  --temperature 0.8 \
  --top-k 20
```

The script prints the generated caption.

## 7. Text-Only Baseline

This repository also includes a text-only GRU language model baseline. The baseline uses only captions and does not receive image input.

Train the baseline:

```bash
python src/train_text_baseline.py \
  --data-dir data/processed \
  --epochs 5 \
  --batch-size 16 \
  --lr 1e-3
```

The baseline can be used to compare image-conditioned generation against text-only generation.

## 9. Notes on Empty Captions

After only one epoch of training, the decoder may sometimes generate an empty caption because it predicts the end-of-sentence token too early. To reduce this issue during inference, the generation script supports:

```bash
--min-len
--temperature
--top-k
```

For example:

```bash
python src/generate_caption.py \
  --image data/raw/instagram_data/img/insta11974.jpg \
  --checkpoint outputs/checkpoints/clip_caption_best.pt \
  --min-len 5 \
  --temperature 0.8 \
  --top-k 20
```

More training epochs usually improve generation stability.

## 13. Project Summary

This project implements a trainable multimodal caption generation pipeline. It uses CLIP for visual representation and trains a GRU decoder to generate Instagram-style captions. The model is small enough for a course project but still demonstrates the key components of modern multimodal text generation.
