# VisualRedNoteGen: Multimodal Social Media Caption Generation

## 1. Project Overview

VisualRedNoteGen is a multimodal text generation project that generates social-media-style captions from images. The project uses a pretrained CLIP image encoder to extract visual features from an input image, and trains a GRU-based caption decoder to generate captions token by token.

The goal of this project is to explore whether visual information can be used to guide social-media caption generation. Instead of generating text from a prompt only, the model receives an image as input and produces an Instagram-style caption conditioned on the visual content.

This project satisfies the **Multimodal Modeling** extra criterion because the generated text is directly conditioned on image input.

## 2. Dataset

This project uses the **Instagram Images with Captions** dataset from Kaggle.

After downloading and unzipping the dataset, the expected raw data structure is:

```text
data/raw/
  instagram_data/
    captions_csv.csv
    img/
      insta1.jpg
      insta2.jpg
      insta3.jpg
      ...
````

The CSV file contains image-caption pairs. In this dataset, the important columns are:

```text
Image File
Caption
```

Example rows contain image paths such as:

```text
img/insta1
img/insta2
```

The preprocessing script automatically resolves these paths to the corresponding `.jpg` image files.

The raw image files are **not included** in this GitHub repository because they are large. Please download the dataset separately from Kaggle.

## 3. Model Architecture

The main model is a **CLIP-conditioned caption generator**.

The architecture is:

```text
Input image
   ↓
Pretrained CLIP image encoder
   ↓
Image embedding
   ↓
Trainable projection layer
   ↓
Initial hidden state of GRU decoder
   ↓
Generated caption tokens
```

The CLIP image encoder is frozen during training. The trainable components are:

1. Image-to-hidden projection layer
2. Token embedding layer
3. GRU caption decoder
4. Output vocabulary projection layer

The decoder is trained using next-token prediction with cross-entropy loss.

During training, each caption is converted into a sequence:

```text
<bos> token_1 token_2 ... token_n <eos>
```

The model receives the image and previous caption tokens, then predicts the next token at each time step.

---

## 4. Extra Criterion

This project pursues the **Multimodal Modeling** extra criterion.

The model uses both image and text modalities:

* **Image input:** processed by the pretrained CLIP image encoder
* **Text output:** generated by the trainable GRU decoder

The image representation directly initializes the decoder hidden state, so this is not a text-only language model.

### Step 1: Clone the Repository

```bash
git clone <repo-url>
cd VisualRedNoteGen
```

### Step 2: Create Environment

```bash
conda create -n visualrednote python=3.11 -y
conda activate visualrednote
pip install -r requirements.txt
```

### Step 3: Download Dataset

Download the Instagram Images with Captions dataset from Kaggle and place it under:

```text
data/raw/instagram_data/
```

Expected files:

```text
data/raw/instagram_data/captions_csv.csv
data/raw/instagram_data/img/
```

### Step 4: Preprocess the Dataset

Run:

```bash
python src/prepare_data.py \
  --csv data/raw/instagram_data/captions_csv.csv \
  --image-root data/raw/instagram_data \
  --image-col "Image File" \
  --caption-col "Caption"
```

This creates:

```text
data/processed/train.csv
data/processed/val.csv
data/processed/test.csv
data/processed/vocab.json
```

The processed split uses approximately:

* 80% training data
* 10% validation data
* 10% test data

The vocabulary is built from the training captions.

## 8. Train the CLIP-Conditioned Caption Model

For a quick local test:

```bash
python src/train_caption.py \
  --data-dir data/processed \
  --epochs 1 \
  --batch-size 4 \
  --lr 1e-3 \
  --clip-model-name openai/clip-vit-base-patch32
```

For the main experiment:

```bash
python src/train_caption.py \
  --data-dir data/processed \
  --epochs 5 \
  --batch-size 16 \
  --lr 1e-3 \
  --clip-model-name openai/clip-vit-base-patch32
```

The trained checkpoints are saved to:

```text
outputs/checkpoints/clip_caption_best.pt
outputs/checkpoints/clip_caption_last.pt
```

The best checkpoint is selected based on validation loss.

## 9. Training Results

The CLIP-conditioned caption model was trained for 5 epochs.

| Epoch | Train Loss | Train PPL | Val Loss | Val PPL |
| ----- | ---------: | --------: | -------: | ------: |
| 1     |     5.6633 |    288.09 |   4.9984 |  148.18 |
| 2     |     4.5770 |     97.22 |   4.7929 |  120.65 |
| 3     |     3.7569 |     42.81 |   4.8029 |  121.86 |
| 4     |     3.0103 |     20.29 |   4.9316 |  138.60 |
| 5     |     2.3891 |     10.90 |   5.1154 |  166.56 |

The best validation checkpoint was saved at **epoch 2**.

After epoch 2, the training loss continued to decrease, while the validation loss increased. This suggests that the model began to overfit the training captions. This is expected because Instagram captions are noisy, informal, and sometimes only weakly aligned with the image content.

Therefore, the final evaluation and generation examples use:

```text
outputs/checkpoints/clip_caption_best.pt
```

## 10. Evaluate the Model

Run:

```bash
python src/evaluate_caption.py \
  --data-dir data/processed \
  --split test \
  --checkpoint outputs/checkpoints/clip_caption_best.pt
```

Final test result:

```text
test_loss: 4.7662
test_perplexity: 117.47
```

This confirms that the trained multimodal model can generalize to the held-out test set at a similar level to the validation set.

## 11. Generate Captions

Generate a caption for one image:

```bash
python src/generate_caption.py \
  --image data/raw/instagram_data/img/insta11974.jpg \
  --checkpoint outputs/checkpoints/clip_caption_best.pt \
  --min-len 5 \
  --temperature 0.8 \
  --top-k 20
```

The generation script supports:

```text
--min-len       minimum number of tokens before allowing <eos>
--temperature  sampling temperature
--top-k         top-k sampling size
```

These options help avoid empty captions and make the generated captions more diverse.

## 13. Discussion

The model successfully learns the general style of Instagram captions. Many generated outputs are short, informal, and emotionally expressive, which is typical of social media text. For example, the model generates phrases such as:

```text
i love this pic!
happy birthday i love you soooo much!!!!
```

However, the model also has limitations:

1. Some captions are generic.
2. Some outputs contain `<unk>` tokens.
3. The generated caption is not always strongly grounded in the image.
4. The model begins to overfit after a few epochs.

These limitations are reasonable for this project because:

* Instagram captions are noisy and informal.
* Some captions are not literal descriptions of the image.
* The decoder is a relatively simple GRU model.
* The vocabulary uses word-level tokenization, which causes rare words to become `<unk>`.

Future improvements could include:

* Using subword tokenization instead of word-level tokenization
* Fine-tuning a pretrained language model decoder
* Using a Transformer decoder instead of GRU
* Adding beam search
* Training for longer with stronger regularization
* Using a larger or cleaner image-caption dataset

## 14. Text-Only Baseline

This repository also includes a text-only GRU language model baseline.

The baseline uses only captions and does not receive image input. It is useful for comparing a normal language model with the multimodal CLIP-conditioned model.

Train the baseline:

```bash
python src/train_text_baseline.py \
  --data-dir data/processed \
  --epochs 5 \
  --batch-size 16 \
  --lr 1e-3
```

The baseline can learn general Instagram caption style, but it cannot adapt the generated caption to the image.

## 15. Notes on Empty Captions

During early experiments, the model sometimes generated empty captions because it predicted the end-of-sentence token too early. This is common in early-stage sequence generation models.

To reduce this issue, the generation function supports a minimum generation length:

```bash
--min-len 5
```

It also supports sampling controls:

```bash
--temperature 0.8
--top-k 20
```

Example:

```bash
python src/generate_caption.py \
  --image data/raw/instagram_data/img/insta11974.jpg \
  --checkpoint outputs/checkpoints/clip_caption_best.pt \
  --min-len 5 \
  --temperature 0.8 \
  --top-k 20
```

## 17. Project Summary

This project implements a complete multimodal text generation pipeline. It uses CLIP as a pretrained visual encoder and trains a GRU decoder to generate Instagram-style captions. The model is simple enough for a course project but still demonstrates the main ideas behind multimodal text generation:

1. Image feature extraction
2. Visual conditioning
3. Sequence generation
4. Next-token prediction training
5. Quantitative evaluation with perplexity
6. Qualitative analysis of generated captions

The final model achieves:

```text
test_loss: 4.7662
test_perplexity: 117.47
```

Overall, VisualRedNoteGen demonstrates a working CLIP-guided caption generation system for social-media-style text generation.
